VisuMatch

Linear Algebra and Geometry Lecture, '23 — VisuMatch is a feature-rich Content Based Image Retrieval (CBIR) system, developed for ITB's Linear Algebra and Geometry Class, utilizing NextJS, React, and Golang to provide an intuitive image search experience based on color and texture with advanced functionalities, including webscraping, camera integration, and report generation.

GoPat

Strategy Algorithm Lecture, '24 — GoPat is a web tool that uses BFS and IDS algorithms to find the shortest path between Wikipedia URLs, leveraging Go-colly for web scraping. Built with Next.js, Go, and Tailwind, it allows users to compare algorithm performance and visualize the shortest path between articles.

NextJS React Golang Gin Gonic Docker Tailwind PostgreSQL TypeScript

Fingerprint Matcher

Strategy Algorithm Lecture, '24 — This project is a fingerprint matching application that utilizes the Knuth-Morris-Pratt (KMP) and Boyer-Moore (BM) algorithms to search for pattern matches. If no match is found, the Levenshtein Distance algorithm is used to find the image with the highest similarity percentage. The application is built using .NET and runs on Docker with MySQL as the database. Additionally, the application can be connected to a local MySQL database as an alternative to Docker.

.NET SDK WPF Docker

Cyberpunk 2077 Breach Protocol Solver

Strategy Algorithm Lecture, '24 — The project is a web-based and CLI application built with JavaScript/TypeScript and Next.js to solve the Cyberpunk 2077 breach protocol problem using a brute-force algorithm. In this hacking mini-game from Cyberpunk 2077, players must find a buffer of tokens in a matrix of tokens, constrained by a maximum buffer length and a specific token sequence. The program aims to find the most optimal buffer with the shortest length and the highest reward based on the provided sequences. The web version allows users to input data in three ways: manually, randomly, or through a file, and the solution can be downloaded as a text file. A CLI version is also available, with input and output handled via the test folder
